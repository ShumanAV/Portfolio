Проект №1 - вебприложение для учета книг в библиотеке, работа с БД реализовано с помощью Spring JDBC Template. В БД две таблицы: Люди и Книги, связь Один-ко-Многим. 
	   В приложении реализован следующий фукнционал, можно:
	   	- просмотреть список всех людей, посмотреть отдельного человека, добавить нового человека, изменить или удалить существующего человека
		- просмотреть список всех книг, посмотреть отдельную книгу, добавить новую книгу, изменить или удалить существующую кнгу
		- назначить книгу человеку, когда человек ее берет
		- освободить книгу
		- в отдельном человеке посмотреть какие книги он взял

Проект №2 - вебприложение для учета книг в библиотеке, работа с БД с помощью Spring Data JPA, а также отдельно на Spring Boot. В БД есть 2 таблицы: Люди и Книги, связь Один-ко-Многим. 
	В приложении реализован следующий фукнционал, можно:
	   	- просмотреть список всех людей, посмотреть отдельного человека, добавить нового человека, изменить или удалить существующего человека
		- просмотреть список всех книг, посмотреть отдельную книгу, добавить новую книгу, изменить или удалить существующую кнгу
		- в отдельной книге можно посмотреть свободна ли она, или какой человек ее взял, назначить книгу человеку, когда человек ее берет, либо освободить книгу
		- в отдельном человеке посмотреть какие книги он взял, какие из них просроченные, какие нет
		- на странице поиска книг можно искать книги по первым символам названия книги, также на странице поиска книг, можно посмотреть свободна ли данная книга, и если нет, то
		какой человек ее взял
		- просмотреть список всех книг с учетом пагенации и сортировки, количество книг на одной странице пагинации и сортировка настраивается

Проект №3 - имитация метеостанции, взаимодействие клиента (датчиков температуры/дождя) и сервера, передача результатов измерений на сервер, реализовано REST API. 
	В БД две таблицы: сенсоры и измерения. 
	В приложении реализован следующий фукнционал:
		- реализован сервер, принимающий два Post и два Get запросов, а именно: 
			- Post запросы: 
				1. клиент присылает имя сенсора для его регистрации в системе в виде json файла (имя сенсора), если такого сенсора не найдено, новый сохраняется в БД
				2. клиент присылает поток новых измерений в виде json файлов (температура, наличие дождя, название сенсора), осуществляется запись измеений в БД
			- Get запросы: 
				1. клиенту отправляется список всех измерений из БД в виде json файла
				2. клиенту отправляется количество дождливых дней в виде числа
		- реализован клиент, отправляющий и получающий данные запросы
		- на клиенте при помощи библиотеки Xchart строится график полученных с сервера температур
